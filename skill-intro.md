### accessibility_testing

- **用途**: 进行无障碍（Accessibility）审计。
- **能力**: 识别 WCAG 合规性问题，检查 ARIA 实现，验证键盘导航和屏幕阅读器兼容性。
- **适用范围**: 审查代码的无障碍性、ARIA 实现或屏幕阅读器兼容性时。

### agent_orchestration

- **用途**: 编排和管理多 Agent 协作。
- **能力**: 将澄清后的用户请求转化为针对特定专家 Agent（如 CTO、AI 架构师）的委托提示词。
- **适用范围**: 在需求澄清完成后，将任务分发给专家 Agent 之前。

### algorithmic-art

- **用途**: 使用代码创作艺术。
- **能力**: 使用 p5.js 创建带有随机种子的生成艺术、流场或粒子系统。
- **适用范围**: 用户要求用代码创作艺术、生成艺术图案时。

### api_design

- **用途**: 设计 API 规范。
- **能力**: 设计 GraphQL Schema、OpenAPI 规范，处理版本控制。
- **适用范围**: API 设计阶段（非实现阶段）。如果涉及代码实现，请使用 `backend_api`。

### app-store-optimization

- **用途**: 应用商店优化 (ASO)。
- **能力**: 研究、优化和追踪移动应用在 Apple App Store 和 Google Play Store 上的表现。
- **适用范围**: 需要提升应用在商店中的排名和转化率时。

### arch_decisions

- **用途**: 制定和记录架构决策。
- **能力**: 提供 ADR（架构决策记录）模板，辅助数据库选型、容量规划和可扩展性设计。
- **适用范围**: 系统设计初期或面临重大技术选型时。

### arch_patterns

- **用途**: 选择和应用架构模式。
- **能力**: 指导单体 vs 微服务、分层架构、事件驱动、CQRS 等模式的选择。
- **适用范围**: 确定系统整体架构风格时。

### audience_intelligence

- **用途**: 目标受众分析。
- **能力**: 分析受众的人口统计学、心理学特征、行为和平台偏好，辅助网红营销。
- **适用范围**: 制定营销策略或选择合作网红时。

### auth_patterns

- **用途**: 实施认证和授权系统。
- **能力**: 掌握 JWT, OAuth2, 会话管理, RBAC 等安全访问控制模式。
- **适用范围**: 构建登录系统、保护 API 或调试安全问题时。

### aws_architect

- **用途**: AWS 解决方案架构设计。
- **能力**: 为初创公司设计无服务器、可扩展且具有成本效益的云基础设施。
- **适用范围**: 规划 AWS 部署架构、DevOps 流程或 IaC 时。

### aws-penetration-testing

- **用途**: AWS 环境渗透测试。
- **能力**: 测试 AWS 安全性，枚举 IAM，利用云基础设施漏洞，S3 桶测试等。
- **适用范围**: 用户要求对 AWS 进行安全评估或渗透测试时。

### backend_api

- **用途**: 后端 API 开发实现。
- **能力**: 实现 REST 接口，验证，安全头配置，认证模式。
- **适用范围**: 编写后端 API 代码时（设计请用 `api_design`）。

### backend_core

- **用途**: Node.js/TypeScript 后端基础。
- **能力**: 规范项目结构，实施 TypeScript 严格模式，遵循最佳实践。
- **适用范围**: 搭建后端项目脚手架或规范现有代码库时。

### backend_database

- **用途**: 数据库交互与优化。
- **能力**: 实施 Repository 模式，处理事务，缓存策略及查询优化。
- **适用范围**: 编写数据库访问层代码或进行性能调优时。

### backend-dev-guidelines

- **用途**: 全面的后端开发指南。
- **能力**: 指导 Node.js/Express/TypeScript 微服务开发，涵盖分层架构、错误处理、验证等。
- **适用范围**: 创建路由、控制器、服务、中间件或进行重构时。

### better_auth

- **用途**: 高级认证系统实现。
- **能力**: 实现登录、注册、OAuth、2FA、MFA、Passkeys 及用户会话管理。
- **适用范围**: 需要构建高安全性、功能完整的认证系统时。

### brainstorming

- **用途**: 创意发散与头脑风暴。
- **能力**: 探索用户意图、需求和设计概念，避免直接进入实现细节。
- **适用范围**: 在任何创造性工作（如功能开发、组件构建）开始前。

### brand-guidelines

- **用途**: 应用品牌设计规范。
- **能力**: 将 Anthropic 的官方品牌颜色和排版应用到文档或设计中。
- **适用范围**: 创建需要符合特定品牌调性的工件（Artifacts）时。

### cache_patterns

- **用途**: 缓存策略实施。
- **能力**: 在 Spring Boot 应用中启用和操作 Spring Cache 抽象层。
- **适用范围**: 优化应用性能，实施应用级缓存时。

### canvas-design

- **用途**: 创建视觉设计工件。
- **能力**: 生成海报、艺术品或其他静态视觉设计的 PNG/PDF 文档。
- **适用范围**: 用户请求创建设计图、海报或艺术作品时。

### claude-d3js-skill

- **用途**: D3.js 数据可视化。
- **能力**: 创建交互式图表、网络图、地理可视化等复杂 SVG 可视化。
- **适用范围**: 需要超越标准图表库的定制化数据可视化时。

### clerk_auth

- **用途**: Clerk 认证集成。
- **能力**: 集成 Clerk 进行现代化认证，支持 WebAuthn, Passkeys 和社交登录。
- **适用范围**: 使用 Clerk 作为身份验证提供商时。

### cli_developer

- **用途**: 命令行工具开发。
- **能力**: 使用 Python (Typer), Go (Cobra), Rust (clap) 构建专业的 CLI 工具，处理参数解析和交互。
- **适用范围**: 开发开发者工具、自动化脚本或系统管理 CLI 时。

### code_formatter

- **用途**: 代码格式化与规范。
- **能力**: 配置 Prettier/ESLint，确保代码风格一致性。
- **适用范围**: 设置项目代码规范或整理代码风格时。

### code_review

- **用途**: 代码审查（Code Review）。
- **能力**: 审查 PR，检测代码异味（Code Smell），检查最佳实践。
- **适用范围**: 审查代码时（交付物验证请用 `quality-validator`）。

### cold_email

- **用途**: 编写冷启动邮件（Cold Emails）。
- **能力**: 生成包含 A/B 测试、跟进时机建议和社交证明的 7-14 封邮件序列。
- **适用范围**: 需要进行对外邮件营销或销售线索挖掘时。

### competitor_analysis

- **用途**: 竞品分析。
- **能力**: 进行 SEO 和策略对比，分析市场定位。
- **适用范围**: 分析竞争对手并制定差异化策略时。

### competitor_intelligence

- **用途**: 竞争对手情报收集。
- **能力**: 分析竞争对手的 SEO/GEO 策略、关键词排名和反向链接概况。
- **适用范围**: 寻找超越竞争对手的市场机会时。

### compliance_analyst

- **用途**: 合规性分析。
- **能力**: 研究认证标准、合规要求和监管路径。
- **适用范围**: 涉及行业标准（如 GDPR, HIPAA）或法律合规性时。

### content-creator

- **用途**: 内容创作与营销。
- **能力**: 创建 SEO 优化的营销内容，分析品牌声音，生成社交媒体帖子。
- **适用范围**: 撰写博客、社交媒体内容或规划内容日历时。

### content_strategist

- **用途**: 内容策略规划。
- **能力**: 创建在搜索引擎中排名靠前的高质量内容，进行关键词优化和结构规划。
- **适用范围**: 制定能够带来流量的内容营销策略时。

### context_engineering

- **用途**: 上下文优化工程。
- **能力**: 优化 Agent 的上下文容量，降低 token 消耗和延迟。
- **适用范围**: 处理长上下文任务或优化 Agent 系统性能时。

### core-components

- **用途**: 核心组件库开发。
- **能力**: 使用设计系统模式构建 UI 组件库和 Token。
- **适用范围**: 构建基础 UI 组件或维护设计系统时。

### data_transform

- **用途**: 数据转换与 ETL。
- **能力**: 使用 SQL (dbt), Python (pandas/polars) 构建 ETL/ELT 管道。
- **适用范围**: 构建数据管道、迁移数据或进行复杂数据清洗时。

### data_visualization

- **用途**: 数据分析与绘图。
- **能力**: 分析数据结构并选择最佳可视化类型，生成柱状图、折线图等。
- **适用范围**: 需要将数据转化为直观图表时。

### database_design

- **用途**: 数据库设计。
- **能力**: 设计 Schema，制定迁移策略，优化索引和查询。
- **适用范围**: 数据库建模或性能调优阶段。

### database_migration

- **用途**: 数据库迁移。
- **能力**: 执行零停机迁移，处理数据转换和回滚程序。
- **适用范围**: 修改数据库结构或在不同平台间迁移数据时。

### debugging_methodology

- **用途**: 系统化调试方法论。
- **能力**: 应用“复现-隔离-假设-修复”的系统化循环来解决 Bug。
- **适用范围**: 遇到复杂 Bug 需要科学排查时。

### debugging_tools

- **用途**: 调试工具使用。
- **能力**: 熟练使用 Console, Debugger, Network tab, Profiling 工具。
- **适用范围**: 进行具体的技术层面的调试操作时。

### deploy_cicd

- **用途**: 部署与 CI/CD 自动化。
- **能力**: 编写 GitHub Actions 工作流，配置 Vercel/Railway 部署，管理发布。
- **适用范围**: 设置自动化构建、测试和部署流程时。

### deploy_monitoring

- **用途**: 部署监控。
- **能力**: 配置健康检查、指标收集、报警系统和回滚策略。
- **适用范围**: 确保生产环境服务的稳定性和可观测性时。

### deps_npm

- **用途**: 依赖管理。
- **能力**: 管理 npm/yarn 依赖，遵循 package.json 最佳实践。
- **适用范围**: 添加、更新或审核项目依赖库时。

### design_components

- **用途**: 组件设计规范。
- **能力**: 定义按钮、卡片、输入框等组件的尺寸和样式规则。
- **适用范围**: 统一 UI 组件视觉标准时。

### design_patterns

- **用途**: UI 设计模式。
- **能力**: 处理视觉层级、Z 轴索引、阴影、动画和留白。
- **适用范围**: 优化界面视觉效果和交互细节时。

### design_responsive

- **用途**: 响应式设计。
- **能力**: 运用断点、流式排版、容器查询等现代 CSS 特性。
- **适用范围**: 确保应用在不同设备上完美展示时。

### design_tokens

- **用途**: 设计变量（Tokens）管理。
- **能力**: 定义 8 点网格、排版阶梯和色彩系统基础变量。
- **适用范围**: 建立设计系统的基础样式变量时。

### dispatching-parallel-agents

- **用途**: 并行任务分发。
- **能力**: 识别可并行执行的独立任务并分发给多个 Agent。
- **适用范围**: 面对多个无依赖关系的独立任务时。

### doc-coauthoring

- **用途**: 文档协作撰写。
- **能力**: 指导用户通过结构化流程共同撰写提案、技术规范等文档。
- **适用范围**: 与用户合作起草复杂文档时。

### docker_optimization

- **用途**: Docker 镜像优化。
- **能力**: 使用多阶段构建、层缓存等技术减小镜像体积并提高构建速度。
- **适用范围**: 优化容器部署效率和安全时。

### docs_api
